# Project Dossier Template

This template defines the structure for project dossiers. Content for each section is determined by the interactive workflow in SKILL.md — do not add content guidance here.

---

```markdown
# Project Dossier

## [Project ID] [Project Name]

# Project Overview

[Narrative paragraph — scope determined by Phase 2 interactive session]

Client Name: [From source materials or user input]

Project Objective: [From source materials or user input]

Target Audience: [If applicable to project type — omit for internal/technical projects]

Duration: [Known dates only, or "To be determined during detailed planning"]

Budget: [If provided, or "To be determined during detailed planning"]

# Expected Outcomes

[Measurable success criteria — how the client or team will know this project succeeded.
Include for ALL project types. Source from materials or interactive session.]

# Project Strategy

[Include this section ONLY for marketing, communications, brand, or outreach projects.
Omit entirely for technical, software, or consulting projects.]

## Key Messages
[Core messaging themes]

## Channels
[Distribution or execution channels]

# Deliverables

[CRITICAL: Organize by area of concern, not timeline phase.
Each area becomes a Task List in Teamwork import.]

### [Deliverable Name]
[Narrative paragraph describing the deliverable's role in the project]

- Ownership: [Team or person responsible]
- Type: [One-time / Recurring — if recurring, specify frequency]
- Dependencies: [Other deliverables or external inputs required]

- Due date: [Only if provided by user — otherwise omit entirely]

### [Deliverable Name]
[Continue for each deliverable in this area]

[Meeting cadence, tools, reporting structure]

[Touchpoints, approval process, key contacts, escalation path]

## [Risk Name]
```
